Spectrum Design Write-up

Jesse Trippe

Critique

  • Side-nav is visually dominant - both because of icons and because of dark color.
  • Icons are hard to understand without instruction or definition.
  • Lots of unused space due to both side-nav AND topbar.
  • Overall color-scheme is slightly washed out.
  • Because of washed-out color-scheme, some text is slightly difficult to scan/read.
  • Client site link, dashboard, and feedback seem out of place.
  • Table design is big, friendly, and usable.
  • Actions-icons are repetitive and hard for a new user to understand.

UX Considerations & Decisions

  • Removed side-nav in favor of top-nav.
    • Shortened page names.
    • Moved help to upper right-hand corner.
    • Add user avatar for friendliness and personalization.
    • Moved feedback to bottom of the page.
    • Moved settings inside of user dropdown.
    • Moved sign-out inside of user dropdown.
    • Added responsive menu that pushes extra menu items into overflow menu (could use usage data to determine which apps go into overflow menu).
    • Move to hamburger menu on mobile and remove username.
  • Adjusted overall page colors for higher-contrast and making the actual content (in this case user rows) more readable.
  • Increased font-size for legibility/readability.
  • Moved action items into a dropdown
    • Frees up a lot of space in the table
    • Allows for descriptions of action instead of just icon
    • Thought about moving to a Gmail-type approach where hovering over the row revealed the button icons in the row (possibly overlayed on top of other cells to maintain space-savingings) but ultimately found the added benefit of text actions (instead of icons) to tip my choice to a dropdown. Could possibly prototype and test with users or A/B test.
  • Moved total count near pagination to be closer to relevant information.

Technology

  • For this project, and many others, I use a SASSed version of SuitCSS http://suitcss.github.io/ to get started. I then stole bits and pieces from Bootstrap to fill in the gaps. Traditionally, I would steal more from Bootstrap's JS libraries but I thought I'd create some interaction JS to show-off basic jQuery/JS skills.
  • I use Gridlover http://gridlover.net/ to create some basic typography rules.
  • I used Entypo for icons, and Icomoon to build an SVG defs file.
  • I used jQuery for interactions. Would normally try to write vanilla JS but...
  • I used jQuery DataTables for table interactions - search/filter, column-sorting.

Opportunities

  • Match Spectrum's UI typography.
  • Would like to hear more about mobile usage stats to determine the best approach. Does Spectrum have a mobile app? If not, how often are users using the Contacts Manager on a mobile device? When they are, how/why are they using it? Is browse and filter still important or is finding a specific user and contacting them most important?
  • Add tooltips to the Initial Contact icons to explain what they mean.
  • Add subtle UI interaction animations.
  • Write more accessible, semantic HTML.
  • Spend more time with aligning icons - specifically in cells and buttons. Current implementation will not scale.
  • More cross-browser testing. Currently built for Mac Chrome, quickly tested in Mac Safari, Opera, and Firefox.
